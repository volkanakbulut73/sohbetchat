
POCKETBASE KURULUM VE KOLEKSİYON REHBERİ
========================================

Uygulamanın çalışabilmesi için PocketBase Admin Paneli (http://127.0.0.1:8090/_/) üzerinde
aşağıdaki "Collections" (Koleksiyonlar) manuel olarak oluşturulmalıdır.

Geliştirme aşamasında kolaylık olması için API Rules kısımlarını boş (public) bırakabilirsiniz.

1. KOLEKSİYON: registrations
   Type: Base
   Fields:
   - nickname (Text)
   - full_name (Text)
   - email (Email)
   - password (Text)
   - criminal_record_file (Text) -> (Base64 string saklayacağı için Text seçin)
   - insurance_file (Text) -> (Base64 string saklayacağı için Text seçin)
   - status (Select) -> Options: pending, approved, rejected

2. KOLEKSİYON: messages
   Type: Base
   Fields:
   - sender (Text)
   - text (Text)
   - type (Text)
   - channel (Text)

3. KOLEKSİYON: channels
   Type: Base
   Fields:
   - name (Text)
   - islocked (Bool)

   * Örnek Veri Ekleme (Channels):
     "New Record" diyerek name="#Sohbet" ve islocked=False olan bir kayıt ekleyin.

4. KOLEKSİYON: system_config
   Type: Base
   Fields:
   - key (Text)
   - value (Text)

   * Örnek Veri Ekleme (Admin Girişi İçin):
     1. Kayıt: key="admin_username", value="admin"
     2. Kayıt: key="admin_password", value="password123"
     3. Kayıt: key="bot_personality", value="Sen mIRC botu Lara'sın..."

5. KOLEKSİYON: notifications_log
   Type: Base
   Fields:
   - type (Text)
   - target (Text)
   - subject (Text)
   - body (Text)
   - sender_admin (Text)

API RULES (İZİNLER)
-------------------
Sağ üstteki "Edit Collection" -> "API Rules" sekmesinden:
- List/Search action: Boş bırakın (Public) veya "@request.auth.id != ''"
- View action: Boş bırakın (Public)
- Create action: Boş bırakın (Public)
- Update action: Boş bırakın (Public)
- Delete action: Boş bırakın (Public)

Not: Prodüksiyon ortamında bu kurallar sıkılaştırılmalıdır.
